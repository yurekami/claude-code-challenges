# Challenge Categories

This document details the 6 challenge categories, their learning objectives, and the Claude Code tips they cover.

## 1. CLI Fundamentals

**Difficulty**: Easy
**Related Tips**: 0-10

Master the basics of Claude Code including configuration, navigation, and essential commands.

### Topics Covered
- Status line customization (Tip 0)
- Slash commands: `/usage`, `/chrome`, `/mcp`, `/stats`, `/clear` (Tip 1)
- Voice input and local models (Tip 2)
- Problem decomposition (Tip 3)
- Git and GitHub CLI integration (Tip 4)
- Fresh conversation strategies (Tip 5)
- Output extraction methods (Tip 6)
- Terminal aliases (Tip 7)
- Basic input methods (Tip 10)

### Challenges
1. **Status Line Setup** - Configure status line with model, directory, git, tokens
2. **Slash Command Explorer** - Master essential slash commands
3. **Git Integration Basics** - Use Claude Code with git workflows
4. **Alias Configuration** - Set up productivity aliases
5. **Output Extraction** - Extract outputs to clipboard, files, URLs

---

## 2. Context Management

**Difficulty**: Medium
**Related Tips**: 5, 8, 13, 23

Learn to effectively manage Claude Code's 200k token context window.

### Topics Covered
- Fresh start strategies (Tip 5)
- Context compaction and HANDOFF.md (Tip 8)
- Conversation history search (Tip 13)
- Fork and clone operations (Tip 23)

### Challenges
1. **Context Compaction** - Manual compaction with preserved context
2. **Handoff Documents** - Create effective HANDOFF.md files
3. **Session Recovery** - Find and resume previous sessions
4. **Fork Strategies** - Branch conversations effectively

---

## 3. MCP Integrations

**Difficulty**: Medium-Hard
**Related Tips**: 11, 12, 20, 25

Extend Claude Code capabilities through Model Context Protocol servers.

### Topics Covered
- Gemini CLI fallback (Tip 11)
- Custom workflows and CLAUDE.md (Tip 12)
- Notion as bridge tool (Tip 20)
- Skills vs Commands vs Plugins (Tip 25)

### Challenges
1. **MCP Server Setup** - Configure Playwright MCP
2. **Custom CLAUDE.md** - Optimize default prompts
3. **Skill Creation** - Build custom skills
4. **Multi-Tool Workflows** - Integrate multiple MCP servers

---

## 4. Testing & Verification

**Difficulty**: Medium
**Related Tips**: 9, 28, 34

Master test-driven development and verification workflows.

### Topics Covered
- tmux test patterns (Tip 9)
- Verification through tests, Git, self-checking (Tip 28)
- Test-driven development (Tip 34)

### Challenges
1. **tmux Test Pattern** - Autonomous test execution
2. **Git Bisect Debugging** - Find breaking commits
3. **TDD Workflow** - Test-first development with Claude
4. **Output Verification** - Systematic result checking

---

## 5. Workflow Automation

**Difficulty**: Medium-Hard
**Related Tips**: 14, 16, 17, 29, 41

Optimize productivity through workflow automation.

### Topics Covered
- Terminal cascade method (Tip 14)
- Git worktrees for parallel work (Tip 16)
- Exponential backoff patterns (Tip 17)
- DevOps and CI/CD debugging (Tip 29)
- Automation of automation (Tip 41)

### Challenges
1. **Terminal Cascade** - Multi-session organization
2. **Git Worktrees** - Parallel branch work
3. **CI/CD Integration** - Debug GitHub Actions
4. **Progressive Automation** - Build automation systems

---

## 6. Advanced Orchestration

**Difficulty**: Hard
**Related Tips**: 21, 31, 36

Master complex multi-agent and containerized workflows.

### Topics Covered
- Container sandboxing (Tip 21)
- Universal interface paradigm (Tip 31)
- Subagent and background execution (Tip 36)

### Challenges
1. **Container Sandbox** - Safe execution environments
2. **Subagent Army** - Coordinate multiple agents
3. **Multi-Model Orchestra** - Orchestrate different AI models
4. **Background Execution** - Long-running process management
